Bakery Clicker

Projet mené en équipe : d’abord identifier les besoins, structurer le travail avec un cahier des charges clair, puis développer le jeu en Python.

  • Python
  • Mode projet
  • Cahier des charges
  • Travail d’équipe
Code (GitHub)
Découvrir le projet

01 — Cadrage

Identifier les besoins

En amont du développement, nous avons commencé par clarifier ce dont le projet avait besoin : objectifs pédagogiques, contraintes techniques (desktop, Python), règles de jeu, livrables attendus et répartition des rôles dans l’équipe.

Cette phase a permis d’aligner tout le groupe sur le même périmètre avant d’écrire une ligne de code.

02 — Planification

Cahier des charges structuré

Les activités ont été planifiées à partir d’un cahier des charges propre : fonctionnalités attendues, architecture modulaire, assets graphiques, jalons et critères d’acceptation pour valider chaque étape.

Ce document a servi de référence commune pour coder, tester et livrer le jeu de façon cohérente.

03 — Suivi de projet

Indicateurs et écarts

Lorsque un suivi de projet a été mis en place, nous avons évalué les indicateurs (avancement, tâches restantes, respect des jalons) et analysé les écarts par rapport au planning initial pour réajuster le travail en équipe.

  • Besoins Recenser le périmètre, les contraintes et les livrables avant le développement.
  • CDC Cahier des charges structuré : fonctionnalités, jalons et critères d’acceptation.
  • Suivi Indicateurs de suivi et analyse des écarts, lorsque le pilotage du projet a été effectué.

04 — Réalisation

Jeu 2D desktop : logique Python, assets et interface issus du cadrage et du CDC.

Projet Python — démarche projet avant développement (cadrage, CDC, suivi).
